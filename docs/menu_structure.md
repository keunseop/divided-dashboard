# 메뉴 구조 제안 (v1)

Streamlit 사이드바를 일반 사용자 중심으로 재배치하고, 관리자 기능은 별도 섹션으로 묶습니다. 로그인 없이도 접근 제어가 가능하도록 P1에서 관리자 비밀번호 게이트를 추가할 예정입니다.

## 일반 사용자 메뉴

| 표시 라벨 | 페이지 파일 | 설명 |
| --- | --- | --- |
| **대시보드** | `pages/1_대시보드.py` | KRW 기준 주요 카드/차트가 있는 홈 화면 |
| **포트폴리오 가져오기** | `pages/2_포트폴리오_가져오기.py` | 보유 종목 LOT/스냅샷 CSV 업로드 및 매수 기록 |
| **배당 내역 가져오기** | `pages/3_배당_내역_가져오기.py` | Excel 기준 배당 CSV 업로드 및 동기화 |
| **보유 종목 배당 추이** | `pages/4_보유_종목_배당_추이.py` | 즐겨찾기 티커들의 배당 추이 비교 |
| **종목 검색** | `pages/5_종목_검색.py` | 국내/해외 티커별 배당/가격 정보 검색 |
| **알림톡 파서** | `pages/6_알림톡_파서.py` | 알림톡 메시지를 파싱하여 배당 이벤트에 반영 |

> 일반 사용자 메뉴는 좌측 사이드바 상단에 그대로 노출합니다.

## 관리자 메뉴 (비밀번호 게이트 적용 예정)

| 표시 라벨 | 페이지 파일 | 비고 |
| --- | --- | --- |
| **[관리자] 배당 원장 테이블** | `pages/90_관리자_배당_원장_테이블.py` | 전체 배당 데이터 관리용, require_admin 필요 |
| **[관리자] 종목 마스터 관리** | `pages/91_관리자_종목_마스터_관리.py` | Ticker Master CRUD, require_admin 필요 |
| **[관리자] 미등록 티커 확인** | `pages/92_관리자_미등록_티커_확인.py` | CSV 대비 미등록 티커 확인, require_admin 필요 |
| **[관리자] DART 단건 조회(디버그)** | `pages/93_관리자_DART_단건_조회.py` | DART API 수동 조회/캐시, require_admin 필요 |

> 관리자 메뉴는 사이드바의 마지막 섹션으로 배치하고, 버튼 클릭 시 관리자 비밀번호를 요구하는 흐름을 P1에서 구현합니다.

## 요약

1. 좌측 메뉴는 “일반 사용자 → 관리자” 순으로 배치합니다.
2. 각 메뉴명/페이지 타이틀은 모두 한글로 통일합니다.
3. 관리자 메뉴 진입 시 별도의 로그인 없이 비밀번호 게이트를 통해 접근 제어합니다.
